## 五、数据信息流

### 5.1 离线侧：一条行为数据的旅程

以 `U0001, V0258, play, 120, 2025-03-15 14:30:00` 为例：

```
数据源：video_behavior_history.csv
  │
  ├─→ 2_offline_analysis.py
  │     │
  │     ├─→ [RDD] map → reduceByKey → sortBy
  │     │     结果: U0001共53次行为（活跃度排名）
  │     │     结果: V0258被播放13960次（播放量排名）
  │     │
  │     └─→ [Spark SQL] GROUP BY + JOIN
  │           结果: V0258综合热度66166分（热度排行第1）
  │           结果: 美食类总行为31594条（类别分析）
  │           结果: 14点行为4260条（时段分析）
  │           结果: 整体点赞率32.45%（互动率）
  │
  ├─→ 3_recommendation.py
  │     │
  │     ├─→ 行为→评分: play=1, like=3, comment=2, share=4, favorite=5
  │     │     U0001对V0258的评分 = 1+3 = 4分
  │     │
  │     ├─→ 构建评分矩阵: [1000用户 × 500视频]
  │     │
  │     ├─→ ALS训练: 矩阵分解 → 用户特征[20] × 视频特征[20]
  │     │     RMSE = 2.47
  │     │
  │     └─→ 为U0001推荐Top10视频（预测评分最高的10个）
  │
  └─→ 6_visualization.py
        │
        └─→ 读取output中的CSV → matplotlib画图
```

### 5.2 实时侧：一条事件的处理过程

```
4_producer.py                          5_streaming.py
─────────────                          ─────────────
while True:                            读Socket
  │                                      │
  ├─ 随机选用户: U0497                   ├─ 收到一行:
  ├─ 随机选视频: V0192                   │   "U0497,V0192,like,0,2026-06-23 17:42:13"
  ├─ 随机选行为: like                    │
  ├─ 随机观看时长: 0                     ├─ split(",") 拆字段:
  ├─ 获取当前时间: 17:42:13              │   UserID=U0497
  │                                      │   VideoID=V0192
  ├─ 拼成行:                             │   Action=like
  │   "U0497,V0192,like,0,              │   WatchTime=0
  │    2026-06-23 17:42:13"             │   EventTime=2026-06-23 17:42:13
  │                                      │
  ├─ conn.send(line) ───────────────→   ├─ 加时间戳: ts=current_timestamp()
  │                                      │
  │                                      ├─ 15秒窗口聚合:
  │                                      │   window=[17:42:00, 17:42:15]
  │                                      │   窗口内:
  │                                      │     V0192: plays=3, likes=2 → hot=3+2×3=9
  │                                      │     V0258: plays=8, likes=5 → hot=8+5×3=23
  │                                      │     ...
  │                                      │
  │                                      ├─ 排序: V0258(23) > V0192(9) > ...
  │                                      │
  │                                      └─ 输出: [17:42:15] 实时热门 Top5
  │                                           1. V0258 hot=23
  │                                           2. V0192 hot=9
  │                                           ...
  └─ sleep(0.1秒) → 下一轮
```

## 六、数据设计

### 6.1 video_info.csv（视频信息表）

| 字段 | 类型 | 示例 | 说明 |
|------|------|------|------|
| VideoID | string | V0258 | 视频唯一标识，V0001~V0500 |
| Title | string | 减脂餐V0258 | 分类模板词 + VideoID |
| Category | string | 美食 | 10个分类之一 |
| Duration_sec | int | 180 | 视频时长，15~600秒 |
| UploadDate | string | 2025-03-15 | 上传日期，过去一年随机 |
| UploaderID | string | U042 | 上传者，U001~U100 |

**10个分类：** 音乐、游戏、美食、科技、搞笑、教育、体育、生活、时尚、旅行

### 6.2 video_behavior_history.csv（用户行为记录）

| 字段 | 类型 | 示例 | 说明 |
|------|------|------|------|
| UserID | string | U0001 | 用户唯一标识，U0001~U1000 |
| VideoID | string | V0258 | 视频ID |
| Action | string | play | 6种行为之一 |
| WatchTime | int | 120 | 观看时长（秒），非播放行为为0 |
| EventTime | string | 2025-03-15 14:30:00 | 事件时间（精确到秒） |

**6种行为及权重：**
| 行为 | 权重 | 含义 | 评分值 |
|------|------|------|--------|
| play | 55% | 播放 | 1 |
| like | 18% | 点赞 | 3 |
| comment | 10% | 评论 | 2 |
| share | 8% | 分享 | 4 |
| favorite | 6% | 收藏 | 5 |
| follow | 3% | 关注 | 2 |

### 6.3 模拟数据的设计思路

1. **用户偏好：** 每个用户随机偏好2-4个分类，70%的行为集中在偏好分类（模拟真实兴趣）
2. **视频热度：** 使用Zipf分布（幂律分布），少数视频爆火，大部分长尾（模拟真实短视频平台）
3. **行为分布：** 播放最常见（55%），关注最稀有（3%），模拟真实转化漏斗
4. **时间分布：** 事件时间随机到日/时/分/秒，保证24小时分析有意义

## 七、推荐算法说明

### 7.1 三种推荐策略

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 热度推荐 | 统计历史播放/点赞等，加权排序 | 简单，不需要训练，新用户友好 | 千人一面，头部集中 |
| ALS协同过滤 | 矩阵分解，学习用户和视频的隐特征向量 | 个性化，能发现潜在兴趣 | 冷启动问题，需离线训练 |
| 用户画像推荐 | 统计用户在各分类的行为，推荐偏好分类的热门视频 | 可解释性强 | 颗粒度粗 |

### 7.2 ALS算法流程

```
用户行为 → 评分矩阵[1000×500] → 矩阵分解 → 用户特征[1000×20] × 视频特征[20×500]
                                              ↓
                               预测评分 = 用户特征向量 · 视频特征向量
                                              ↓
                               选评分最高的Top10推荐给用户
```

**参数：** rank=20（隐因子维度）, regParam=0.01（正则化）, maxIter=10
**评估：** RMSE ≈ 2.47（评分范围0-10）

## 八、实时处理说明

### 8.1 架构

使用**Spark Structured Streaming**（基于DataFrame的流处理API），数据源为Kafka（已实际部署并测试通过），同时保留Socket模式作为备选。

### 8.2 三个并发查询

| 查询 | 用途 | 窗口 | 输出模式 |
|------|------|------|----------|
| hot_videos | 实时热门视频排行 | 15秒窗口，5秒滑动 | Complete（内存表） |
| action_stats | 行为分布统计 | 15秒窗口，5秒滑动 | Complete（内存表） |
| console | 控制台实时输出 | 15秒窗口，5秒滑动 | Complete（控制台） |

### 8.3 窗口计算示意

```
时间轴:  0    5   10   15   20   25   30
         ├────┼────┼────┼────┼────┼────┤
窗口1:   [0──────────15)
窗口2:        [5──────────20)
窗口3:            [10──────────25)
窗口4:                 [15──────────30)

15秒窗口，每5秒滑动一次，相邻窗口有10秒重叠
```

### 8.4 Kafka 配置

项目已完整部署 Kafka 3.9.2（Zookeeper 模式）：

- Zookeeper 端口：12181
- Kafka Broker 端口：9092
- Topic：video_events（1分区，1副本）
- 数据格式：CSV（UserID,VideoID,Action,WatchTime,EventTime）
- Producer 速率：10条/秒

**启动 Kafka：**
```bash
# 终端1：启动 Zookeeper（如未启动）
cd D:\Spark\kafka_2.13-3.9.2
export JAVA_HOME="D:/Spark/jdk17/jdk-17.0.19+10"
bin/zookeeper-server-start.sh config/zookeeper.properties

# 终端2：启动 Kafka Broker
bin/kafka-server-start.sh config/server.properties
```

**切换回 Socket 模式：** 将 `4_kafka_producer.py` 中 `KAFKA_AVAILABLE` 和 `5_streaming.py` 中 `USE_KAFKA` 改为 `False`。
